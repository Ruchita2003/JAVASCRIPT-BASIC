Que-1 What is JavaScript?


Ans- JavaScript is a high-level, dynamic, interpreted programming language that is primarily used to add interactivity and dynamic behavior to web pages. Que-5 What are undeclared and undefined variables?

Ans-Undeclared variables: These are variables that have been used in code without being formally declared using the var, let, or const keywords. When you attempt to use an undeclared variable, JavaScript will either create an implied global variable (in non-strict mode) or throw a reference error (in strict mode). Using undeclared variables is generally considered bad practice because it can lead to unexpected behavior and difficult-to-debug issues. Undefined variables: These are variables that have been declared but have not been assigned a value. In this case, the variable exists, but it does not yet hold any meaningful value. Accessing an undefined variable typically results in the value undefined.

Que-8 What is === operator?

Ans-The === operator is a strict equality operator in JavaScript. It checks whether the two operands are both equal in value and data type. It returns true if the operands are strictly equal and of the same type, and false otherwise. For example, 5 === 5 would return true because both the value and the type are the same, whereas 5 === '5' would return false because the types are different.

Que-18 What are the disadvantages of using innerHTML in JavaScript?

Ans- While the innerHTML property is a convenient way to manipulate the content of HTML elements in JavaScript, it has some disadvantages that developers should be aware of: 1. Security risks: Using innerHTML can expose your application to security vulnerabilities such as cross-site scripting (XSS) attacks. If the content being inserted via innerHTML includes user-generated or untrusted data, it could potentially execute malicious scripts embedded in that data. 2. Performance implications: Manipulating innerHTML can be inefficient, especially when dealing with large amounts of HTML content. When you use innerHTML, the entire content of the element is serialized and parsed again, which can impact performance, especially on complex or large pages. 3. Overwriting existing event handlers: When you use innerHTML to modify the content of an element, it can potentially overwrite any event listeners or data associated with the existing elements. This can lead to unexpected behavior and bugs in your application. 4. Loss of previously bound references: If you have JavaScript references to specific elements within the replaced content, those references may become invalid after using innerHTML to modify the content. This can result in unexpected behavior and errors in your code.
